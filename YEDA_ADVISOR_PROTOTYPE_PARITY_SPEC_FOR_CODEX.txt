## Target flow

### Step 1 — Real questionnaire

The current questionnaire remains the real input source.

Do not replace it with a fake prototype form.

The user fills the existing form.

The submit action should not immediately call the backend. Instead:

1. Validate/build the payload exactly as today.
2. Store the payload in frontend state.
3. Move to the preference summary screen.

### Step 2 — Preference summary

Show a premium summary screen before backend call.

Cards should summarize real form values:

- תקציב
- שנתון
- שימוש עיקרי
- סוג רכב
- דלק / גיר
- מספר מושבים / משפחה
- סגנון נהיגה
- משקלי העדפות:
  - אמינות
  - דלק
  - שמירת ערך
  - ביצועים
  - נוחות

Buttons:

- primary: `נתח את ההעדפות שלי`
- secondary: `חזור לעריכה`

Clicking primary starts the existing backend request with the saved payload.

### Step 3 — AI analyzing screen

While the existing backend request is running, show a premium analysis screen.

Visual behavior:

- centered glass/chrome panel
- AI/spinner/pulse animation
- clear status steps
- optional ETA/timing banner integrated visually
- no ugly raw technical loading state

Status steps:

- `סורק התאמה לתקציב ולשנתון`
- `מצליב אמינות ועלויות תחזוקה`
- `בודק צריכת דלק / חשמל`
- `מחשב עלות בעלות שנתית`
- `מדרג התאמה אישית`

### Step 4 — Results

Render the real backend results exactly as before, but inside the premium shell.

Must include:

- top recommendation hero card
- circular score gauge
- top car key facts:
  - price
  - annual cost
  - monthly estimate if safely calculated
  - fuel/energy
  - reliability/maintenance indication
  - safety
- result cards for all cars
- warnings/methodology preserved

Do not remove data to make the UI shorter.

### Step 5 — Internal compare and details

These are same-page views, not full page reloads.

The existing standalone `/compare` page must remain untouched and functional.

#### Compare view

Compare the top available cars from current results:

- 1 car: hide or disable compare with explanation
- 2 cars: compare 2
- 3+ cars: compare top 3

Compare rows:

1. התאמת AI
2. טווח מחיר
3. עלות שנתית כוללת
4. עלות חודשית משוערת
5. צריכת דלק / חשמל
6. סיכון תחזוקה / אמינות
7. ביטוח שנתי
8. אגרת רישוי שנתית
9. בטיחות
10. שמירת ערך
11. ביצועים
12. נוחות ואבזור
13. היצע בשוק

Winner highlighting:

- Higher is better for fit, reliability, safety, resale, comfort, performance.
- Lower is better for annual cost, monthly cost, fuel/energy cost, insurance, license fee, maintenance.
- Do not mark a winner when the comparison is textual or unsafe.

#### Details view

For a selected car, show:

- back to results
- score gauge
- car title
- full cost breakdown
- monthly estimated total
- annual estimated total
- 3-year estimated total
- segmented cost bar
- cost segment rows
- risk notes
- missing-data warnings

## Cost breakdown rules

Use only fields already returned by the advisor result.

Likely available fields include:

- `total_annual_cost`
- `maintenance_cost`
- `annual_energy_cost`
- `annual_fuel_cost`
- `insurance_cost`
- `annual_fee`
- `price_range_nis`
- `resale_value`

Breakdown logic:

1. Fuel/energy:
   - prefer `annual_energy_cost`
   - fallback to `annual_fuel_cost`

2. Insurance:
   - use `insurance_cost`

3. Maintenance:
   - use `maintenance_cost`

4. Licensing:
   - use `annual_fee`

5. Depreciation / residual:
   - calculate only if enough safe data exists.
   - If `total_annual_cost` exists and known components exist, compute residual as `פחת / שונות משוערת` only if residual is positive and reasonable.
   - If residual is negative or suspicious, label as `שונות / חוסר התאמה בנתונים`.
   - Do not present residual as certain depreciation unless the backend provides a direct reliable depreciation field.

6. Monthly total:
   - if `total_annual_cost` exists, divide by 12.
   - otherwise sum known annual components / 12 and label as partial.

7. 3-year total:
   - annual total * 3, or monthly total * 36.
   - label as estimate.

Important:

- If the total is partial, show `חישוב חלקי לפי שדות זמינים`.
- If data is missing, show `לא זמין` and list missing fields.
- Do not invent values.
